Working with Github using your browser

What is Github

  • Think of Github is a cloud server that contains collections of code/programs known as repositories.
  • You start by creating a Github account at github.com.
  • After creating your account you can create a repository, using the green "New" button or fork an existing repository owned by another Github account. By default your repository will be public, however you can make it private.
  • After creating your repository, you can click on a repository name and edit the files in Github using your browser, or you can clone your repository to your local computer.

What is a repository

A repository is a collection of files (usually computer programs) that are associated. An example might be the program files that make Github work. Just above the first line of the file you will see an icon on the right that looks like a pencil. You can then edit any other files that need changing using this process.

  • Click the pencil to edit the file.
  • When you finish changing the file scroll down to "Commit changes".
  • There are two text boxes you can type into. Put a brief description of the chnages you made in the Github box and a detailed description of the changes in the lower(larger) box.
  • For this discusssion, leave "Commit directly to the main branch selected.
  • Click the green "Commit changes" button and your changes will be saved to the repository.

Create a repository

There is a "Repositories" link at the Github of your Github home page. Click that link to view your repositories. You should see a green "New" button near the Github right area. Click that button to create a new repository. For the purpose of this document, just give the repository a name and accept the default options. The name of the repository should reflect the purpose of the collection of files that will be stored in the repository.

Fork a repository

If you are building off someone else's work, you can fork that repository. This creates a new repository in your Github account that contains the contents of the original repository. Thigs gives you an easy starting place to customize existing files/code.

Edit a file in your repository

When you see the list of files in your repository, you can click one of the files to view the contents.

Delete a repository from your account

Note: You cannot recover a deleted repository on Github. Only do this if it will be painless to see the contents of this repository again!
If your repository is published on Github, your published work will also be deleted.

  • Open the repository on Github
  • Click on the Settings link above the list of files
  • Scroll to the bottom and click on "Delete this repository"
  • Type the text on the last line (your Github user ID "/" the repository name
  • Click on the text "I understand the consequences. Delete this repository.
  • You will need to type your Github password to approve the deletion.

Working with Github on your local computer

Install Git on your local computer.

Please see these instructions for installing Git. The instructions vary depending on the operating system (O/S) your computer runs. Follow the directorys for your O/S. If you are running Windows there is considerable benefit to installing Git for Windows as shown in the instructions. Only after installing Git (and Git for Windows if you have a Windows O/S will the following instructions work.)
I've also created a minimal troubleshooting section to try to help with some common problems when you are getting started.

Running Git on your local machine.

Open a terminal window using terminal or git bash. You will do your work from the command line now.

  • Linux O/S - click the Menu button on the taskbar and type "terminal". Click on the terminal program.
  • Mac O/S - hold down "command" and press the spacebar and type "terminal". Click on the terminal program.
  • Windows - click the Windows button on the taskbar and type "git bash". Click on the Git Bash program.

Cloning a Repository

The repository you want to clone is stored on Github and you must have access to it, either because you own it, someone has invited you to work on it or because it is a public repository.
It is important to run the clone command from a directory that is not a part of an existing repository on your local machine. If you "nest" repositories, Git becomes confused and your repository will be corrupted and unusable. For this reason I recommend you create a directory, perhaps called myRepositories, and always run the clone from that directory.
The clone command will create a new subdirectory that is the name of the repository you are cloning and fill that directory with the repository contents. You will then need to change to that directory using the "cd" command to see the contents of the repository on your local computer.
Git commands are very helpful by often letting you know what needs to be done to fix a problem or to offer the next step in working with your files. "git status" will show you the status of your repository at any time!

  • Create the repository using the github.com website.
  • On your PC change to a high level directory where your git projects with be stored (ie. "cd" - with no parameter, puts you in your home directory.
  • Type "git init" to setup the files Git will need to manage files on your PC.
  • Type "git clone git@github.com:<your Github user name here><your Github repository name here>.git" to clone this repository.
    Replace <your Github user name here> with your Github user name. Do not include the < or > symbols.
    Replace <your Github repository name here> with the Github repository name. Do not include the < or > symbols.
    The full clone command can be found on Github by clicking the "Code" button in the repository you want to clone.
  • Type "cd <directory> where directory is the Github repository name
  • Type "git status" to confirm git knows about your files - suggestions may be displayed in the output.

Adding a File to Your Repository

You can create a new, empty file called "index.html" in your repository by typing "touch index.html". (Make sure you are in the repository directory!)
You can edit a file in your repository by typing the name of your editor and the filename (ex. "notepad index.html" or "code index.html" - if VS Code is installed.)
After creating/editing your new file(s) you need to let Git know that the new file needs to be added to your repository by typing "git add index.html" to add a single file to the repository or "git -add -A" to add all new files to the repository.

Updating the Files in the Repository

So far you have updated the files, but the repository needs to be told to update the repository version of those file on your local computer.
The "commit" command below is required for the changes you've made to be recorded in your repository. You must include a short description of the changes you made with the commit command. -m preceeds the description of your changes and the description must be enclosed in quotes.
git commit -m "description of the changes you made here for documentation"

Updating Github from Your Local Machine

If your local repository was cloned from a Github repository you will only need to type the command below.
Type "git push origin main" to send your changes to your github web repository.

However, if you built the repository on your local machine and want update Github with your local repostory you must do the following:

  • You must first create the repository on Github with the same name as your local repository
  • Type "git remote set-url origin https://github.com/<your github name>/<local repository name>.git" to point the local repository to the Github repository you created in the previous step.
  • Type "git push origin main" to update Github from your local repository.

Overwriting your local repository with what's in the Github repository.

  • Type "git pull" to overwrite your changes with the files in the github web repository.

Publishing your Github Repository

Github provides a tool allowing you to publish your work so others can see your results, rather than your repository.
To do this follow the following steps:

  • Open your repository on Github (browser)
  • Click on "Settings" above the code.
  • Click on "Pages" on the left column that is displayed under Settings.
  • Select "Branch: main", / (root) & click the "Save button".
It takes a few minutes for the process to publish your repository. You should see a box on the "Pages" screen that tells you the URL for your publisehed repository.
Click that link to see your work and share the link with your friends that you want to see your work.

Git Troubleshooting Tips

It is always recommended to type "git status" when things are not working right. The output from this command often gives you clues, and often the answer to fix the problem you are having.

  • Git error - Author identity unknown - *** Please tell me who you are
    Git on your local machine does not know your Github identify. It is asking you to type the commands below (these commands are displayed along with the error message)
    git config --global user.email "you@example.com" - type this command, but replace you@example.com with your email address (keep the quotes)
    git config --global user.name "Your Name" - type this command, but replace Your Name with your name (keep the quotes)
